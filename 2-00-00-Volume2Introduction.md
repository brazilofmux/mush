# Introduction to The MUSH User's Manual

Welcome to *The MUSH User's Manual*, the companion volume to *The MUSH
Standard*. Where the Standard defines how a MUSH server shall behave, this
Manual teaches you how to use one.

Whether you are a brand-new player logging in for the first time, a builder
constructing a sprawling virtual world, a programmer automating game systems,
or an administrator keeping the lights on, this book has something for you.

## Who This Book Is For

This book assumes no prior experience with MUSHes or text-based virtual
worlds. Part I starts from zero: what a MUSH is, how to connect, and how
to talk to people. Each subsequent part builds on the last, introducing
building, programming, community tools, and administration at a comfortable
pace.

Experienced MUSH users will find the later parts valuable as well. The
programming chapters cover techniques from beginner through advanced, and
the administration chapters address topics rarely documented anywhere else.

## How This Book Is Organized

**Part I: Getting Started** covers the basics. You will learn what a MUSH
is, how to connect to one, how to communicate with other players, and how
to navigate the virtual world.

**Part II: Building Your World** teaches you to create rooms, exits,
objects, and players. You will learn to describe your creations, set
attributes, and use locks to control access.

**Part III: MUSHcode Programming** introduces the MUSH programming language.
Starting with simple expressions, you will progress through variables, control
flow, string manipulation, list processing, and user-defined commands and
functions. The final chapter covers advanced patterns used by experienced
coders.

**Part IV: Community and Communication** covers the channel system, the
mail system, and bulletin board systems -- the tools that hold a MUSH
community together.

**Part V: Administration** is for server administrators. You will learn to
run a MUSH server, manage players, maintain the database, and secure your
installation.

**Part VI: Reference** provides a quick reference card and a troubleshooting
guide for common problems.

## Conventions Used in This Book

Commands you type are shown in code blocks:

```
> say Hello, world!
You say, "Hello, world!"
```

The `>` prompt indicates what you type. Lines without the prompt show what
the server displays in response.

Object names, flag names, attribute names, and function names appear in
code font: `THING`, `DARK`, `DESC`, `strlen()`.

Tips, warnings, and important notes are called out in the text where they
appear.

## About the Standard

This book frequently references *The MUSH Standard* (Volume 1 in this
series). The Standard is the formal specification of MUSH server behavior.
When this Manual says "the server does X," the Standard is the authoritative
source for exactly how X works.

You do not need to read the Standard to use this Manual. But if you want
to understand *why* something works the way it does, or if you are building
tools that need precise behavioral guarantees, the Standard is there for
you.
